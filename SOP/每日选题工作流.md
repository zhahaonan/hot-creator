# 每日选题工作流 SOP

## 概述

每天固定时间运行，完成从热点采集到内容策划的全流程。

## 触发方式

**自然语言**：对 Agent 说"开始今日选题"
**CLI 命令**：`python scripts/start_my_day.py`

## 流程步骤

### Step 1：采集（自动，约2-5分钟）

- [ ] 热榜数据采集（微博/抖音/知乎/百度）
- [ ] RSS 订阅抓取（36氪/少数派/HackerNews）
- [ ] 合并去重

**产出**：`output/{date}-merged.json`

### Step 2：趋势分析（自动，约1-2分钟）

- [ ] AI 评分（0-100）
- [ ] 跨平台去重
- [ ] 分类标注（科技/财经/娱乐/教育等）
- [ ] 趋势方向判断（上升/顶峰/下降/萌芽）

**产出**：`output/{date}-trends.json`

### Step 3：创作简报（自动，约3-5分钟）

- [ ] 创作角度生成（3-5个角度/话题）
- [ ] 各平台内容大纲（短视频/小红书/公众号）
- [ ] 标题建议
- [ ] 对标案例推荐

**产出**：`output/{date}-briefs.json`

### Step 4：知识库更新（自动，<1分钟）

- [ ] 追加今日话题到知识库
- [ ] 计算跨天关联
- [ ] 更新主题标签频次

**产出**：`output/knowledge_base.json`（累积）

### Step 5：导出（自动，约1分钟）

- [ ] Obsidian 笔记（Dashboard + 单篇话题笔记 + 自动链接）
- [ ] Excel 报表
- [ ] 力导向关系图谱（HTML，含累积历史）

**产出**：
- `HotCreator/{date}/_Dashboard.md`
- `HotCreator/{date}/Topics/*.md`
- `output/hot-creator-{date}.xlsx`
- `output/hot-creator-mindmap-{date}.html`

## Step 6：人工筛选（5-10分钟）

- [ ] 打开 Dashboard，浏览今日话题总览
- [ ] 根据自身领域筛选 2-3 个值得做的话题
- [ ] 打开对应 Topic 笔记，选定创作角度
- [ ] 决定首发平台和发布时间

## Step 7：执行创作

- [ ] 根据选定角度和大纲开始创作
- [ ] 参考标题建议选择最终标题
- [ ] 按推荐时间发布

## 最佳实践

1. **固定时间**：建议每天 8:00-9:00 运行，在当天内容创作前完成
2. **不贪多**：每天聚焦 2-3 个话题，深度胜过广度
3. **关注萌芽**：emerging 话题竞争小，抢先发布优势大
4. **回看图谱**：每周看一次累积图谱，找到持续热点做系列内容
5. **复盘积累**：发布后记录数据，参见[[爆款复盘模板]]

## 跳过采集模式

已有数据时可快速运行分析+导出：

```bash
python scripts/start_my_day.py --skip-collect --skip-analyze -i output/briefs.json
```
